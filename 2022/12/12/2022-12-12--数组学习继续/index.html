数组学习继续

今天想打开我的静态网页，发现怎么都打不开。尝试了一些方法，解决了PR_CONNECT_RESET_ERROR的这个问题：在cmd终端中输入ipconfig/flushdns就可以了。
但是不知道为什么打开网页的速度要慢一些。

1.数组的集成初始化： int a[]={1,2,3,5,9,45,12}; int a[23] = {2}; //输出2 0 0 0 0 0 0 0 ....0 对于C99，还可以指定位置进行初始化： int a[20]={ [0]=2,[4]=7,8, }; 2.数组大小：sizeof(a)/sizeof(a[0]) 3.数组的赋值：直接使用 int b[]=a;这是不行的！！！ 需要采用遍历来赋值。 4.数组作为参数传入函数时，需要使用另一个参数传递大小。例如： int f(int a[],int len){} //这个样子，不能用sizeof算个数了。因为只是传了一个地址吧。 5.在linux系统中，可以直接终端输入 man 函数名 来看这个函数要怎么使用，需要包含什么头文件之类的。有点像matlab中的help。 6.在课程举的例子中，是用素数这个例子来说的。 以前寻找素数的时候，就是按照循环来，按照素数的定义，素数不能整除它前面的数。这样需要做n-1次循环。 其实还可以把所有偶数去除，这样只要循环约n/2次。还可以用n的平方根作为一个界限，这样运算次数会更少。 引入数组之后，就可以使用一种“表”的方式，来扩充数组。只要让这个数除以素数表的数，判断是不是它们的倍数，然后把这个“是”的答案填充进素数表。 甚至，可以利用排除法，比如25以内的素数，依次以倍数的方式排除。 这样的方法和定义和人的思维不同。 我觉得课程有句话说的很有启发性：算法不一定和人的思考方式相同。 我之前其实潜意识里是觉得，我只是用另一种语言方式来做我思考的事情，实际上这一点应该得到扩充，人的思考方式是最适合人的，机器的思考方式，我只能尽量去探索，不一定能找到最合适的。 7.二维数组 int a[3][5]; //3行5列。遍历的时候使用2重循环。 8.二维数组的列数一定要给出的。 int a[][3] = {{1,2,3},{1,2,3}}; 而且做定位的时候需要用两个[][]。 但是实际上存储二维数组的时候，是按照行来变成一维的，应该是这样。
