今天学习的东西是判断

1.判断的语句形式：

if(OK){……..}else{} if(){}else if(){}else{}

就是说，这个else if我之前的理解就是一种固化的写法形式，但是翁老师说它可以以
if(){}else{
if(){}else{}
}
这个样子理解，这就是级联。确实，有else{if}和没有{}是一样的，都是其中一个程序块会被运行。
哇！所以一开始的思路都很简单，很基础，没有一点点复杂的。很奇妙。
2.关系运算符：
==
!=

= （同下）
<
<= (小于或等于，注意是或关系)

2.1 关系运算符比算数运算符(+ - * /)的优先级低，比赋值(=)高.
2.2 它的运算结果是整数 0 和 1
2.3 关系运算符之间的优先级：从左到右/== !=要比其他的低.

  1. ==的话是不是就可以与 同或 同样的效果了，比如：5>3==6>4
    因为==优先级低，所以先算两边的不等式，同真同假的话才可以输出1.话说程序语言中有专门的同或运算嘛？我听的都是异或啊。
    度娘查了一下，异或的逻辑运算符是 ^ ，同或没找着。不过按位的运算和普通的判断条件还是有区别的吧。

4.有一个比较大小的例子
比较a b的大小，好了，输入；输出max

第一种: 第二种：
int max = 0; int max = b;
if(a >b) max = a; if(a > b) max = a;
else max = b;

第一种确实很普适，大家一眼就可以看懂。所以还是很提倡的吧。
但是我看到第二种的时候感觉，额，眼前一亮？因为我自己是不会想到这种写法的。它给我的思路是，以后或许可以试着先预设一种结果，然后依靠这种 类似完备性 这样子再去判断修正结果。这样出来的东西也许很好玩。

  1. switch-case
    注意这个东西是从头到尾的执行。如果无视case是全部顺序的，case是一个执行的开始而已。
    switch(t整型int){
    case 1：…; break;
    case 2(整数，常量const)：…; break;
    default：…;
    }

int main()
{
int x;
scanf(“%d”,&x);

int f = 0;
switch(x<=0){
    case 1: f = -(x!=0);break;
    default: f = 2*x;
}

printf("%d",f);

return 0;

}
